CSC Project Log
===============

2026-03-22: Project initialized
- Read Self-Compressing Neural Networks paper (Csefalvay & Imber 2023) in full
- Key technical details extracted:
  * Quantization: q(x,b,e) = 2^e * round(min(max(2^(-e)*x, -2^(b-1)), 2^(b-1)-1))
  * Loss: L = L_task + gamma * Q, where Q = (1/N) * sum(z_l)
  * Layer size: z_l = I*H*W * sum(b_i) for output channels i
  * Improved eq5 with cross-layer coupling
  * Bias handling: L1 on biases of zero-bit channels, remove only when bias~=0
  * Optimizer: Adam, lr=0.5/1e-3 (quant/weights), eps=1e-3/1e-5, L2=5e-4 on weights
  * Training: 850 iterations then ReduceLROnPlateau
  * gamma sampled log-uniformly from [1e-3, 10^-0.5]
  * Baseline CIFAR-10: 95.69 +/- 0.22%, matches with 3% bits / 18% weights
  * Architecture: Page 2019 fast CIFAR network (shallow ResNet)
- Environment: RTX 4090 (25.8 GB), PyTorch 2.10+cu128, WSL2

- Built complete codebase (see files for details)
- Downloaded datasets: CIFAR-10, CIFAR-100, MNIST to E:/datasets/
- All unit tests pass

EXPERIMENT 11: PackNet (prune=0.75, retrain=10 epochs)
- Status: COMPLETED (re-done with correct implementation)
- Previous result was INVALID (used per-task model snapshots — not a real CL method)
- Corrected implementation: mask-based eval (apply_mask zeros future+pruned weights),
  per-task BN state save/restore (running stats + affine params),
  frozen weight force-restore after each optimizer step (prevents weight decay drift),
  head gradient protection for non-current tasks
- Bugs found and fixed:
  1. Weight decay was modifying frozen weights (~0.09 drift per step)
  2. BN learned affine params (weight/bias) not being saved/restored per task
  3. Head weights for non-current tasks not protected from gradient updates
  4. Free weights (mask==0) not zeroed during eval (were zero when task trained)
  5. Original PackNet assigns free weights to current task before training
     (make_finetuning_mask), our initial impl kept them as mask==0
- Correct Results: Avg Acc 87.18%, BWT 0.00%, Capacity 94.4%
  Per-task: T0=71.3, T1=85.2, T2=84.4, T3=85.4, T4=91.0,
            T5=88.0, T6=90.5, T7=93.8, T8=92.8, T9=89.4
- True zero forgetting confirmed (each row identical in accuracy matrix)
- Task 0 lowest (71.3%) because only 25% capacity retained after pruning
- Later tasks higher (88-93%) due to transfer from frozen earlier features

EXPERIMENT 13: CSC weight-level compression (gamma=0.001, replay=200)
- Status: COMPLETED
- Results: Avg Acc 68.99%, BWT -20.59%, Bits 25.0%
- No compression (gamma too low for weight-level, even more quant params)

GRANULARITY COMPARISON:
  gamma=0.001:
    Channel: 71.00% acc, 20.5% bits (compression happening)
    Group:   70.35% acc, 25.0% bits (no compression)
    Weight:  68.99% acc, 25.0% bits (no compression)
  gamma=0.01:
    Channel: 69.67% acc, 8.7% bits  (strong compression)
    Group:   69.71% acc, 24.8% bits (minimal compression)
    Weight:  70.11% acc, 25.0% bits (no compression)

  KEY FINDING: Channel-level is the only granularity that achieves meaningful
  compression. Group-level barely compresses even at gamma=0.01 (24.8% vs 8.7%).
  Weight-level doesn't compress at all.

  This is because: channel-level has O_l parameters controlling I*H*W weights each.
  The compression gradient per param is proportional to I*H*W (large). For group-level,
  each param controls group_size=16 weights, so the gradient is 16x weaker. For
  weight-level, each param controls 1 weight, so the gradient is I*H*W times weaker.
  The optimizer needs much higher gamma to overcome this.

  Despite no compression, accuracy is similar across granularities (~69-71%), confirming
  the quantization doesn't hurt accuracy and the CL performance comes from replay.

EXPERIMENT 15: CSC gamma=0.0001 (channel-level, replay=200)
- Status: COMPLETED

FULL GAMMA SWEEP (channel-level, replay=200):
  gamma=0.0001 -> 69.84% acc, 24.6% bits
  gamma=0.0005 -> 69.47% acc, 22.7% bits
  gamma=0.001  -> 71.00% acc, 20.5% bits  <-- BEST
  gamma=0.005  -> 69.75% acc, 9.5% bits
  gamma=0.01   -> 69.67% acc, 8.7% bits
  Replay-only  -> 71.42% acc, 100% bits

  gamma=0.001 is the sweet spot: best accuracy while achieving 5x compression.
  Lower gamma (0.0001, 0.0005) = less compression but also slightly LESS accuracy.
  This suggests the compression at gamma=0.001 is acting as a beneficial inductive
  bias — it's not just "not hurting" accuracy, it's slightly helping by forcing
  more efficient representations.

KEY FINDINGS:
1. Soft protection using bit-depth as learned importance signal is the key innovation.
   It provides substantial forgetting resistance while maintaining flexible capacity.
2. Soft CSC b=1.0 r=200 (76.12%) beats replay-only r=500 (76.03%) — achieving the
   same accuracy with 2.5x less replay memory and 4x less model capacity.
3. Soft CSC b=5.0 r=500 (80.11%) is 4% above replay-only r=500 at 1/4 the bits.
   The forgetting (-8.93%) is halved compared to replay-only (-14.07%).
4. Hard freeze doesn't work for CSC — the BN mismatch problem means frozen weights
   need per-task BN, which adds complexity. Soft protection avoids this entirely.
5. The paper story: "Self-compression learns which representations are important
   (high bit-depth). Using this as a continuous importance signal for gradient
   scaling provides implicit forgetting resistance without hard freezing."

COMPRESSION-AS-IMPORTANCE-ONLY (critical reframing experiment)
================================================================================

Setup: Bit-depths are learned via compression loss and used for soft protection,
BUT quantization is REMOVED from the forward pass. Model stays at 100% capacity.
Any improvement over replay-only can ONLY come from the learned importance signal.

RESULTS (Split CIFAR-100, r=200):
  Method                       | 10-task | 20-task | Bits
  -----------------------------|---------|---------|------
  Replay-only                  | 71.42%  | 77.51%  | 100%
  Importance-only g=0.001      | 76.80%  | 81.73%  | 100%  (+5.4%, +4.2%)
  Importance-only g=0.01       | 76.21%  |   -     | 100%
  Importance-only + DER++      | 78.42%  | 82.32%  | 100%  (+7.0%, +4.8%)
  Soft CSC (with quantization) | 76.12%  | 81.42%  | 24.5%

THIS IS THE KEY RESULT. The compression objective discovers importance that
improves CL by 5.4% even WITHOUT any capacity reduction. The model stays at
full size. The improvement is purely from importance-based gradient scaling.

Importance-only (76.80%) is slightly BETTER than full CSC (76.12%) because
quantization overhead slightly hurts. But CSC gives you both: importance AND
compression. The choice is: importance-only for max accuracy, full CSC for
capacity efficiency.

This directly addresses the reviewer concern: "the learned signal is barely
better than random." It IS better — not by 0.3% (the random vs learned gap)
but by 5.4% over replay-only (the gap that matters). The random scaling
ablation showed that heterogeneous LR helps; this experiment shows that
the specific heterogeneity from compression helps MORE than no heterogeneity.

Importance-only with refresh every 5 tasks: 81.51% (slightly below 81.73%)
-> Accumulated importance is more useful than fresh discovery

layer weights serve all tasks and can't be cleanly separated. This is a known
limitation of PackNet for single-head benchmarks, not a bug in our implementation.

CLASSICAL CL BASELINES: EWC & MAS (2026-04-09)
================================================================================

Added two canonical continual-learning baselines, carefully verified against
their source-code publications to avoid implementation bugs:

EWC (Kirkpatrick et al. 2017 + Continual World 2021 for RL):
- Fisher computed from log π(a|s) gradient (NOT SAC actor loss — common bug)
- Actor-only regularization (CW shows critic-reg HURTS)
- 10 × 32 = 320 samples minibatched (better than single batch)
- Fisher clipped at 1e-5, accumulated additively across tasks
- λ = 1e4 (CW default for SAC)
- Snapshot θ* BEFORE Fisher computation (canonical ordering)
- Known limitation: empirical Fisher via batch-mean grad (vs true per-sample)
  - Standard approximation in PyTorch ports; CW uses analytic Gaussian Fisher

MAS (Aljundi et al. 2018 + Continual World 2021 for RL):
- Omega = |∂(||mu(s)||² + ||log_std(s)||²)/∂θ|
- Uses pre-tanh mean + log_std, as CW does (not sampled action)
- Actor-only, 10 × 256 = 2560 samples at task boundary
- Additive accumulation, λ = 1e4 (CW default)

Both sanity-tested: penalty = 0 at snapshot, grows with param drift,
accumulates additively across tasks.

EWC BASELINE RESULTS (seed=42, λ=1e4, full5, 500K/task):
  PickCube:    0.97  ← Perfect retention!
  StackCube:   0.00
  PegInsertion: 0.00
  PushCube:    0.00  ← COMPLETELY FAILS to learn new task
  PlugCharger: 0.00
  Average:     0.19

Trajectory:
